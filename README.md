# Hbnb

Hbnb est un projet académique de la Holberton School, réalisé en groupe avec [EmmanuelMOUMBOUILOU](https://github.com/EmmanuelMOUMBOUILOU) et [Dj-kvt](https://github.com/Dj-kvt) pour la partie back-end, puis en individuel pour la partie front-end. Le but de ce projet était de créer une version simplifiée du site Airbnb. Dans ce projet, j'ai occupé le rôle de développeur web full stack.

## Technologies utilisées

Le projet Hbnb repose sur les technologies suivantes :

- **Back-end** : [Flask](https://flask.palletsprojects.com/en/stable/) avec [flask-restx](https://flask-restx.readthedocs.io/en/latest/) pour la création d'API RESTful.
- **Front-end** : `JavaScript` pour une interface utilisateur réactive et moderne.
- **Base de données** : [MySQL](https://www.mysql.com/) pour le stockage structuré des données.
- **Authentification** : `flask-jwt-extended`, `Simple JWT` pour la gestion sécurisée des utilisateurs et des tokens.

## Bilan du projet

**Méthode Agile :**

Grâce à ce projet, j'ai pu mettre en pratique la méthode Agile, avec une répartition des tâches ayant chacune une deadline grâce à Trello, ainsi que des points réguliers avec les autres développeurs. Cela permettait également de réorganiser les tâches si nécessaire.

La répartition des tâches n’a pas été optimale. Nous avions décidé de nous répartir des tâches dans différentes parties du projet afin que chacun puisse apprendre tous les aspects du projet. Cependant, cela pouvait poser problème, car nous pouvions être bloqués si une tâche dépendait de celle d’un autre membre.

Cette expérience m'a permis de comprendre comment mieux répartir les tâches pour mes prochains projets et d’avancer plus rapidement.

Avec du recul, je donnerais toutes les tâches d’un même aspect du projet à un développeur pour éviter les blocages, tout en restant informé du travail des autres pour comprendre ce qui a été fait. Cela aurait été beaucoup plus simple que de mélanger des tâches de tous les aspects du projet.

**Assemblage du travail :**

J'ai également appris à assembler le travail réalisé par chaque développeur pour obtenir un résultat fonctionnel malgré les différentes manières de coder. Cela m'a permis d’analyser le code en détail pour identifier rapidement les problèmes lorsque nous réunissions toutes les contributions.

La partie la plus compliquée a été de gérer mes propres tâches tout en assemblant les contributions sur la branche `develop`, dans un délai assez court.

Avec du recul, il aurait été préférable de terminer d’abord mes tâches, puis de m’occuper ensuite de l’assemblage de l’ensemble du code.

**Les branches/noms de fichiers :**

Concernant les branches, j'avais déjà compris leur utilité et veillais à utiliser des noms explicites. Du côté des fichiers, j’utilisais également des noms explicites.

Dans ce projet, j'ai constaté à quel point les noms explicites de branches et de fichiers sont cruciaux. Ils permettent de gagner du temps pour identifier rapidement un problème et le corriger.

**Conclusion :** Ce projet de groupe m'a énormément appris sur l’organisation, la répartition des tâches, l’assemblage du code de chacun et la mise en place d’une méthode de travail simple et efficace pour mes futurs projets.
